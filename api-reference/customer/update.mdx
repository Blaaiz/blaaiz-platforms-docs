---
title: Update a customer
description: Update customer details and attach documents
api: PUT /api/external/customer/{id}
---

This endpoint allows you to update the details of an existing customer.

## Headers

<ParamField header="x-blaaiz-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the customer
</ParamField>

## Body Parameters

<ParamField body="first_name" type="string">
  Customer's first name
</ParamField>

<ParamField body="last_name" type="string">
  Customer's last name
</ParamField>

<ParamField body="business_name" type="string">
  Business name
</ParamField>

<ParamField body="email" type="string">
  Customer's email address
</ParamField>

<ParamField body="phone_number" type="string">
  Customer's phone number
</ParamField>

<ParamField body="address" type="object">
  Customer's address details
</ParamField>

<ParamField body="id_file" type="string">
  File ID for identity document (obtained from File API)
</ParamField>

<ParamField body="proof_of_address_file" type="string">
  File ID for proof of address document
</ParamField>

<ParamField body="liveness_check_file" type="string">
  File ID for liveness check document
</ParamField>

## Response

<ResponseField name="status" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="object">
  The updated customer object
</ResponseField>

<RequestExample>
```bash cURL - Update Details
curl --request PATCH \
  --url https://api-dev.blaaiz.com/api/external/customer/customer_123456789 \
  --header 'Content-Type: application/json' \
  --header 'x-blaaiz-api-key: YOUR_API_KEY' \
  --data '{
    "phone_number": "+2348098765432",
    "address": {
      "street": "456 New Street",
      "city": "Lagos",
      "state": "Lagos",
      "country": "NG",
      "postal_code": "100001"
    }
  }'
```

```bash cURL - Attach Documents
curl --request PATCH \
  --url https://api-dev.blaaiz.com/api/external/customer/customer_123456789 \
  --header 'Content-Type: application/json' \
  --header 'x-blaaiz-api-key: YOUR_API_KEY' \
  --data '{
    "id_file": "file_123456",
    "proof_of_address_file": "file_789012",
    "liveness_check_file": "file_345678"
  }'
```

```javascript Node.js
const response = await fetch('https://api-dev.blaaiz.com/api/external/customer/customer_123456789', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json',
    'x-blaaiz-api-key': 'YOUR_API_KEY'
  },
  body: JSON.stringify({
    id_file: 'file_123456',
    proof_of_address_file: 'file_789012',
    liveness_check_file: 'file_345678'
  })
});

const data = await response.json();
```

```python Python
import requests

url = "https://api-dev.blaaiz.com/api/external/customer/customer_123456789"
headers = {
    "Content-Type": "application/json",
    "x-blaaiz-api-key": "YOUR_API_KEY"
}
payload = {
    "id_file": "file_123456",
    "proof_of_address_file": "file_789012",
    "liveness_check_file": "file_345678"
}

response = requests.patch(url, json=payload, headers=headers)
data = response.json()
```

<Note>
When attaching documents, use the `file_id` values obtained from the File API after uploading the documents. Once all required documents are attached, the customer status will change from `pending` to `verified`.
</Note>
