---
title: Add KYC details
description: Add KYC (Know Your Customer) details for a customer.
api: POST /api/external/customer/{id}/kyc-data
---

This endpoint allows you to add KYC details for a customer.

By default, a customer's verification status will be PENDING. You need to upload some files for the customer before the status moves to VERIFIED.

## Endpoint

```
POST /api/external/customer/{id}/kyc-data
```

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the customer.
</ParamField>

## Headers

<ParamField header="Accept" type="string" default="application/json">
  Content type header.
</ParamField>

<ParamField header="x-blaaiz-api-key" type="string" required>
  Your API key for authentication.
</ParamField>

## Body parameters

<ParamField body="dob" type="string">
  Date of birth (YYYY-MM-DD format).
</ParamField>

<ParamField body="street" type="string">
  Street address.
</ParamField>

<ParamField body="city" type="string">
  City.
</ParamField>

<ParamField body="zip_code" type="string">
  Zip or postal code.
</ParamField>

<ParamField body="id_expiry_date" type="string">
  ID expiry date (YYYY-MM-DD format).
</ParamField>

<ParamField body="id_issue_date" type="string">
  ID issue date (YYYY-MM-DD format).
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message.
</ResponseField>

<ResponseField name="data" type="object">
  Updated customer object.
  
  <Expandable title="Customer object">
    <ResponseField name="id" type="string">
      Unique identifier for the customer.
    </ResponseField>
    
    <ResponseField name="business_id" type="string">
      Business identifier.
    </ResponseField>
    
    <ResponseField name="first_name" type="string">
      Customer's first name.
    </ResponseField>
    
    <ResponseField name="last_name" type="string">
      Customer's last name.
    </ResponseField>
    
    <ResponseField name="business_name" type="string">
      Business name (if applicable).
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Customer type (individual or business).
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Customer's email address.
    </ResponseField>
    
    <ResponseField name="country" type="string">
      Customer's country code.
    </ResponseField>
    
    <ResponseField name="id_type" type="string">
      Type of identification document.
    </ResponseField>
    
    <ResponseField name="id_number" type="string">
      Identification number.
    </ResponseField>
    
    <ResponseField name="verification_status" type="string">
      Customer verification status.
    </ResponseField>
    
    <ResponseField name="kyc_data" type="object">
      KYC data object.
    </ResponseField>
  </Expandable>
</ResponseField>

```python Python
import requests

url = "https://api-dev.blaaiz.com/api/external/customer/{id}/kyc-data"

    "Accept": "application/json",
    "Content-Type": "application/json",

}

    "dob": "1990-01-01",
    "street": "123 Main St",
    "city": "Lagos",
    "zip_code": "100001",
    "id_expiry_date": "2030-12-31"
}

response = requests.post(url, json=payload, headers=headers)
data = response.json()
```
