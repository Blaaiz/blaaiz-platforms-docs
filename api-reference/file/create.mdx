---
title: Request file upload URL
description: Get a pre-signed URL to upload customer documents
api: POST /api/external/file
---

Request a pre-signed URL to upload customer documents for KYC verification. This is a two-step process: first get the upload URL, then upload the file to that URL.

## Headers

<ParamField header="x-blaaiz-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Body Parameters

<ParamField body="customer_id" type="string" required>
  The unique identifier of the customer
</ParamField>

<ParamField body="file_category" type="string" required>
  The category of the file. Options: `identity`, `proof_of_address`, `liveness_check`
</ParamField>

<ParamField body="file_name" type="string" required>
  The name of the file to be uploaded (including extension)
</ParamField>

<ParamField body="file_type" type="string" required>
  The MIME type of the file (e.g., `image/jpeg`, `image/png`, `application/pdf`)
</ParamField>

## Response

<ResponseField name="status" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="object">
  The upload details
  
  <Expandable title="Upload details">
    <ResponseField name="file_id" type="string">
      Unique identifier for the file (use this when attaching the file to the customer)
    </ResponseField>
    
    <ResponseField name="upload_url" type="string">
      Pre-signed URL where the file should be uploaded
    </ResponseField>
    
    <ResponseField name="headers" type="object">
      Headers to include when uploading the file
    </ResponseField>
    
    <ResponseField name="expires_at" type="string">
      Timestamp when the upload URL expires
    </ResponseField>
  </Expandable>
</ResponseField>

## Upload Process

After receiving the pre-signed URL:

1. Make a `PUT` request to the `upload_url`
2. Include the `headers` from the response
3. Send the file as binary data in the request body
4. Once uploaded, use the `file_id` to attach the file to the customer record

<RequestExample>
```bash cURL - Step 1: Get Upload URL
curl --request POST \
  --url https://api-dev.blaaiz.com/api/external/file \
  --header 'Content-Type: application/json' \
  --header 'x-blaaiz-api-key: YOUR_API_KEY' \
  --data '{
    "customer_id": "customer_123",
    "file_category": "identity",
    "file_name": "passport.jpg",
    "file_type": "image/jpeg"
  }'
```

```bash cURL - Step 2: Upload File
curl --request PUT \
  --url 'UPLOAD_URL_FROM_RESPONSE' \
  --header 'Content-Type: image/jpeg' \
  --data-binary '@/path/to/passport.jpg'
```

```javascript Node.js
// Step 1: Get upload URL
const response = await fetch('https://api-dev.blaaiz.com/api/external/file', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-blaaiz-api-key': 'YOUR_API_KEY'
  },
  body: JSON.stringify({
    customer_id: 'customer_123',
    file_category: 'identity',
    file_name: 'passport.jpg',
    file_type: 'image/jpeg'
  })
});

const { data } = await response.json();

// Step 2: Upload file
const fileBlob = await fetch('/path/to/passport.jpg').then(r => r.blob());

await fetch(data.upload_url, {
  method: 'PUT',
  headers: data.headers,
  body: fileBlob
});

// Step 3: Attach file to customer (see Customer API)
```

```python Python
import requests

# Step 1: Get upload URL
url = "https://api-dev.blaaiz.com/api/external/file"
headers = {
    "Content-Type": "application/json",
    "x-blaaiz-api-key": "YOUR_API_KEY"
}
payload = {
    "customer_id": "customer_123",
    "file_category": "identity",
    "file_name": "passport.jpg",
    "file_type": "image/jpeg"
}

response = requests.post(url, json=payload, headers=headers)
data = response.json()["data"]

# Step 2: Upload file
with open("/path/to/passport.jpg", "rb") as file:
    upload_response = requests.put(
        data["upload_url"],
        headers=data["headers"],
        data=file
    )

# Step 3: Attach file to customer (see Customer API)
```
