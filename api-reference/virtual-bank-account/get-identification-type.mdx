---
title: Get identification type
description: Retrieve the expected identification type label for customer creation
api: GET /api/external/virtual-bank-account/identification-type
---

Retrieve the expected identification type label based on country and customer type. This endpoint returns the label for the `tin` field required when [creating a customer](/api-reference/customer/create#param-tin).

For example, if the customer country is Nigeria, it returns the label "Bank Verification Number", indicating what value should be collected for the `tin` property during customer creation.

## Headers

<ParamField header="x-blaaiz-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Query Parameters

<ParamField query="customer_id" type="string">
  The unique identifier of the customer. If provided, country and type are not required.
</ParamField>

<ParamField query="country" type="string">
  Two-letter country code (e.g., NG). Required if customer_id is not provided.
</ParamField>

<ParamField query="type" type="string">
  The type of customer. Possible values: `business`, `individual`.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Response message.
</ResponseField>

<ResponseField name="data" type="object">
  The identification type details.
  
  <Expandable title="Identification type object">
    <ResponseField name="label" type="string">
      The human-readable label for the identification type (e.g., "Bank Verification Number" for Nigeria).
    </ResponseField>
    
    <ResponseField name="type" type="string">
      The identification type code.
    </ResponseField>
  </Expandable>
</ResponseField>
